# Pitstop — Design Context

> Placeholder project name. Rename freely; this file moves with the project.

## Users
Developers using Claude Code (and other agents that can call MCP tools / hit a small HTTP API). Solo or small-team contexts. Most sessions are mid- to late-stage in a feature task — the agent has done a chunk of work and the developer wants to walk through it before merging or moving on. Often run late in a dev session; eye fatigue is real.

## Brand Personality
**Bracing, exact, dry.** Industrial-feeling without being theatrical. Microcopy reads like terminal output (`PREPARING_03`, `POKE_FAILED · CLICK_RETRY`, `STOP` not "Pause"). Dry humor lives in details you only catch on a second look. Confidently un-cute.

## Aesthetic Direction
Mono Industrial. Dark-first, light supported. Charcoal canvas with subtle dot-grid texture; sans body for legibility, mono for everything else (badges, microcopy, file refs, button labels, status pill, footer counts). Sharp corners. No soft-shadow cards, no side-stripe accents, no glassmorphism, no AI-generic violet pills. Single warm accent (coral/amber, finalized in craft) for "agent is working" states; semantic colors (approved / needs-attention / error) used sparingly.

**Anti-references:** Linear (too generic-dev-tool), Notion (too document-y), GitHub PR review (too review-heavy / multi-stakeholder), shadcn-card aesthetics (too templated).

## Design Principles
1. **Auto-advance is local.** "Looks good" never waits on the agent — the next item is already there. The agent is only invoked when there's something for it to do.
2. **Microcopy carries the voice.** Most personality lives in tiny mono lines and labels, not in chrome decoration. If a label can be drier and still clear, make it drier.
3. **Keyboard-first, mouse-supported.** `j/k`, `⏎`, `c`, `⌘⏎`, `esc`, `?` are the chrome's primary affordance. Visible 2–3 contextual shortcuts in the footer, full keymap on `?`.
4. **No nested cards. No side-stripe accents.** Use typography, hairlines, and the dot-grid frame for structure.
5. **Status is informative, not anxious.** When the agent is working, say what it's doing in mono terms. When something fails, surface a recoverable action inline — never a modal.
